# 阅读体验 + 用户互动 UI 优化设计（My Blog）

日期：2026-03-06  
范围：公共站点（首页、文章页、导航、侧栏、页脚）  
目标：在不牺牲阅读沉浸感的前提下，提升点赞/评论/收藏等互动转化

## 1. 背景与目标

当前站点组件完整、主题体系成熟（OKLCH + 变量化），但存在以下问题：
- 阅读路径与互动路径未形成“主次节奏”，局部区域有信息堆叠感。
- 视觉层级（正文、元信息、互动入口）存在竞争，影响首屏理解效率。
- 评论与互动模块缺少统一的引导策略，用户参与意图未被持续放大。

设计目标（双目标）：
1. 提升阅读体验：降低视觉噪音、增强正文可读性、稳定长文滚动体验。
2. 提升用户互动：优化关键节点 CTA，降低评论参与门槛，提高互动操作意愿。

## 2. 方案选择与结论

候选路径：
1. 阅读优先，互动轻量嵌入（推荐）
2. 互动优先，阅读中强曝光
3. 双轨 A/B 并行

推荐采用路径 1，并在关键位做轻量 A/B（文案与曝光时机）：
- 原因：适合内容型博客长期迭代，风险低，收益稳定。
- 结果：阅读不中断，互动在“合适时机”出现，而非全程干扰。

## 3. 信息架构（已确认）

文章页结构建议：
1. 顶部信息区：标题、作者、发布时间、阅读时长、分类标签（标题优先）。
2. 正文区：主阅读流（可读性第一优先）。
3. 互动区（轻量快捷）：点赞/收藏/分享。
4. 评论引导区：在评论前强化“参与讨论”心智。
5. 评论区：输入优先、结构清晰、空状态友好。
6. 页脚：作为内容结束收尾，不抢主视觉。

## 4. 视觉与交互规范（已确认）

### 4.1 排版与可读性
- 正文容器宽度收紧至阅读友好范围（建议等效 72-78ch）。
- 正文字号 16-18px，正文行高 1.65-1.8。
- 标题层级和段落留白标准化，避免“标题贴正文”。
- 元信息（作者/日期/阅读量）视觉降权，不与标题竞争。

### 4.2 颜色与对比
- 保留现有 OKLCH token，不推翻主题结构。
- 优先修正文案对比度（正文满足 WCAG AA）。
- 互动按钮分层：
  - 主动作：品牌色实底（如发表评论/参与讨论）
  - 次动作：浅底或描边（点赞/收藏）

### 4.3 卡片与边框
- 统一卡片圆角和阴影等级（最多两档）。
- 评论区采用“轻边框 + 留白”替代重阴影堆叠。
- 交互卡片统一 hover/focus 反馈与过渡时长（150-250ms）。

### 4.4 动效
- 保留入场动画，但减少同屏并发动画数量。
- 优先颜色/边框反馈，谨慎使用 scale，避免布局抖动。
- 继续完整支持 `prefers-reduced-motion`。

### 4.5 响应式
- 重点断点：375 / 768 / 1024 / 1440。
- 移动端优先正文与评论输入，侧栏信息后置。
- sticky 导航与首屏内容留白联动，避免遮挡。

## 5. 组件级改造清单与验收标准（已确认）

### 5.1 Navbar
- 改造：降低视觉噪音、稳定 sticky 状态、统一焦点样式。
- 验收：移动端无遮挡；Tab 顺序正确；滚动隐藏/出现平滑。

### 5.2 文章头部
- 改造：标题优先布局，摘要/元信息降权，标签弱强调。
- 验收：3 秒内识别主题、作者、发布时间和主要 CTA。

### 5.3 正文区
- 改造：行宽、行高、段落节奏统一；代码块/引用/表格样式统一壳层。
- 验收：长文滚动疲劳降低；h1-h3 层级清晰；目录锚点稳定。

### 5.4 互动区
- 改造：主次动作分层，不再按钮平铺同权。
- 验收：动作语义清晰；互动模块不抢正文注意力。

### 5.5 评论区
- 改造：输入区优先、登录提示清晰、回复缩进结构强化。
- 验收：评论发布路径直观；回复层级一眼可读；空状态友好。

### 5.6 Sidebar
- 改造：从信息罗列改为任务导向顺序（作者/导航/分类/标签/归档）。
- 验收：用户 2 次点击内可跳转到相关主题内容。

### 5.7 Footer
- 改造：弱化视觉权重，强化实用链接（归档/分类/订阅）。
- 验收：作为阅读收尾，不干扰主体内容。

## 6. 实施节奏（分期）

### 第一期：阅读底盘
- 任务：正文排版、标题层级、目录可读性、导航遮挡处理。
- 指标：平均阅读深度、停留时长提升。

### 第二期：互动提效
- 任务：互动区主次分层、评论引导和空状态优化。
- 指标：点赞率、评论提交率、收藏率提升。

### 第三期：全局一致性
- 任务：Sidebar/Footer 与 token/组件规范统一。
- 指标：跨页面一致性提升、跳出率降低。

## 7. 度量与验证

核心监测指标：
- 阅读：平均停留时长、滚动深度、目录点击率
- 互动：点赞率、收藏率、评论发起率、评论提交率
- 质量：移动端可用性、键盘可访问性、CLS 与 LCP 稳定性

A/B 建议（轻量）：
1. 互动区文案（“文章互动” vs “参与讨论”）
2. 互动区曝光时机（正文后立即 vs 滚动 60% 后强化提示）

## 8. 风险与约束

- 不能为了互动提升而破坏阅读沉浸路径。
- 不能推翻现有主题 token 体系，优先在现有变量上微调。
- 所有交互样式必须同时满足深浅色对比和键盘可访问性。
