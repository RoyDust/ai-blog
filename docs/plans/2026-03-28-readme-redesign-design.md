# README Redesign Design

**Date:** 2026-03-28

## Goal

将仓库首页从“功能罗列型说明”升级为“可展示、可上手、可部署”的项目入口文档，同时确保所有描述都能在当前代码库中找到依据。

## Audience

- 第一次访问仓库的面试官、协作者、潜在使用者
- 需要快速本地启动项目的开发者
- 需要查看部署入口、环境变量和补充文档的维护者

## Inspiration

本次结构参考了优秀开源项目常见的 README 组织方式，重点吸收以下模式：

- 顶部 Hero + 一句话定位 + Badges
- “为什么存在”与“核心亮点”前置
- 功能模块按用户视角分组，而不是单纯堆功能点
- 安装、环境变量、测试、部署路径清晰独立
- 通过补充文档链接把深层信息下沉，而不是把首页写成百科全书

## Content Strategy

README 改为以下结构：

1. Hero 区域
2. 项目简介
3. 核心亮点
4. 功能全景
5. 技术栈
6. 项目结构
7. 快速开始
8. 环境变量说明
9. 测试与质量保障
10. 部署说明
11. 补充文档索引

## Writing Rules

- 优先描述已落地能力，不承诺未来功能
- 用“读者体验 / 创作后台 / 平台能力”组织功能
- 兼顾展示感与可执行性，避免空泛营销词
- 尽量用短段落、表格和命令块提升扫描效率
- 将争议或历史性说明下沉，不在首页展开过多背景

## Accepted Design

- README 主要面向“展示项目质感 + 帮助快速部署体验”的双场景
- 保留中文主文案，便于与项目现有内容风格统一
- 不额外生成截图资源，先通过排版、徽章和信息层级完成包装
