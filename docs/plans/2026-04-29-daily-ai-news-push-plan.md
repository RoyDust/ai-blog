# 每日 AI 新闻推送功能方案

## 目标
在后台新增“AI 新闻推送”能力：每天从固定可信 AI 新闻源抓取近期内容，交给 AI 生成一篇中文优质 AI 新闻日报博客；先创建草稿，再复用现有 AI 审稿能力，审稿通过后自动上线。

## 第一版策略（本次执行）
- 固定 RSS/Atom 新闻源清单写在代码里，不做来源 CRUD。
- 后台提供手动触发入口，便于管理员验证日报质量。
- API 聚合最近 48 小时左右的候选新闻，来源失败时跳过并记录失败信息。
- 按 URL/标题去重，最多交给 AI 20 条候选；AI 负责挑选 8-12 条并生成中文 Markdown 日报。
- 使用确定性 slug：`ai-daily-YYYY-MM-DD`，同一天重复触发不重复创建文章。
- 生成文章先保存为草稿；复用已有 AI 审稿逻辑，通过自动发布门槛后上线，否则保留草稿。
- 不新增依赖、不新增数据库模型、不接入系统定时任务；后续可由外部 cron 调用后台 API。

## 后续阶段
1. 来源管理：后台增删 RSS 源、权重、分类。
2. 调度与历史：持久化运行记录、失败原因、耗时、选中来源。
3. 深度抽取：按 URL 抽取全文，提升文章细节和事实校验。
4. 分发：邮件、飞书/Telegram、RSS 专题推送。
5. 质量控制：重复事件合并、来源白名单信誉分、事实引用完整度评分。

## 验收标准
- 管理员可在后台手动生成今日 AI 日报。
- API 只允许管理员调用。
- RSS/Atom 解析、去重、AI 生成 JSON 解析均有单元测试覆盖。
- 审稿 ready 且分数达标时自动发布；否则保持草稿。
- lint、测试、构建通过。
