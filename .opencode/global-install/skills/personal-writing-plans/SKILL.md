# Personal Writing Plans

## Purpose

Use this after the direction is decided. It turns the chosen approach into a small-step execution plan you can follow without improvising.

## Use When

- A feature or refactor is too large for one quick edit
- You want to avoid missing files, tests, or verification steps
- You already know the direction and now need an implementation path

## Checklist

1. List the exact files likely to change.
2. Break the work into small tasks.
3. Put testing and verification into the plan, not afterthoughts.
4. Keep tasks narrow enough to review and validate quickly.
5. Hand off to `personal-test-driven-development` while implementing.

## Handoff

Load `personal-test-driven-development` and execute the plan incrementally.
