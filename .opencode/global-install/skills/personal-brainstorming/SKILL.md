# Personal Brainstorming

## Purpose

Use this before feature work or meaningful refactors. It converts a rough idea into a clear direction with defined scope and non-goals.

## Use When

- You are adding a feature
- You are changing behavior or UX
- You are refactoring something with architectural trade-offs

## Checklist

1. State the goal in one sentence.
2. Capture constraints, non-goals, and success criteria.
3. Compare two or three approaches.
4. Recommend one approach and explain why.
5. Hand off to `personal-writing-plans` before implementation.

## Handoff

Once the direction is clear, load `personal-writing-plans` and turn the idea into small execution steps.
