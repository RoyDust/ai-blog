# Personal Using Superpowers

## Purpose

Use this skill first. Its job is not to solve the task directly. Its job is to route you into the right workflow so you do not start with the wrong method.

## Use When

- A new task arrives
- You feel tempted to jump straight into editing
- You are unsure whether the task is a feature, bugfix, or refactor

## Checklist

1. Identify the task type: feature, bugfix, refactor, or tiny safe update.
2. Pick the matching personal workflow.
3. Load the next skill before making code changes.
4. Reserve `personal-verification-before-completion` for the end.

## Handoff

- Feature work -> `personal-brainstorming`
- Bugfix -> `personal-systematic-debugging`
- Refactor -> `personal-brainstorming`
- Tiny safe update -> still end with `personal-verification-before-completion`
