# Personal Verification Before Completion

## Purpose

Use this at the end of any meaningful task. It prevents false confidence by requiring evidence before you say the work is complete.

## Use When

- You think the fix is done
- You are about to report success
- You are preparing to commit, open a PR, or hand work off

## Checklist

1. Run the most relevant tests or checks.
2. Run build or lint when the scope justifies it.
3. Do a focused manual check for user-facing changes.
4. Compare the result against the original goal.
5. Report what passed, what you checked, and any remaining risk.

## Handoff

This is the final skill in the loop. If verification fails, return to the appropriate implementation or debugging skill and repeat.
