# Daily Default Workflow

Use this when the task is small or you want the shortest safe routine.

## Flow

1. Load `personal-using-superpowers`.
2. Decide whether the task is feature work, debugging, or refactoring.
3. If the task changes behavior, load the next matching skill instead of improvising.
4. Finish with `personal-verification-before-completion` before claiming success.

## Exit Criteria

- You used the matching workflow instead of jumping straight to edits.
- You verified the result with at least one concrete command or manual check.

## Rule of Thumb

- Feature idea: switch to `feature-flow.md`
- Bug or failing test: switch to `bugfix-flow.md`
- Structural cleanup: switch to `refactor-flow.md`
