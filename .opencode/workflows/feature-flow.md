# Feature Flow

Use this when adding a new feature, changing page behavior, or introducing a new user-facing capability.

## Flow

1. `personal-using-superpowers`
2. `personal-brainstorming`
3. `personal-writing-plans`
4. `personal-test-driven-development`
5. `personal-verification-before-completion`

## Intent

This flow avoids premature coding. You first clarify what should exist, then break it into steps, then implement with tests, then verify with real evidence.

## Exit Criteria

- The feature scope and non-goals are clear.
- The work was broken into actionable tasks.
- The implementation has test coverage where it matters.
- Final verification includes commands or manual checks that prove the feature works.
